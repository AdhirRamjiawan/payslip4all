# Implementation Plan: Serilog File Logging & Global Exception Handling

**Branch**: `002-serilog-logging` | **Date**: 2026-03-19 | **Spec**: specs/002-serilog-logging/spec.md  
**Input**: Feature specification from `/specs/002-serilog-logging/spec.md`

## Summary

Add Serilog as the application's logging provider and implement a `GlobalExceptionMiddleware`
that captures all unhandled exceptions, enriches log entries with request context and user
identity, and writes them to rolling daily log files. Log levels and file paths are fully
configurable via `appsettings.json`. The client always receives a user-friendly response
with no stack trace leakage.

## Technical Context

**Language/Version**: C# 12 / .NET 8 (LTS)  
**Primary Dependencies**: `Serilog.AspNetCore` 8.x (includes `Serilog.Sinks.File`, `Serilog.Settings.Configuration`, enrichers)  
**Storage**: File system вЂ” rolling daily log files at `logs/payslip4all-[date].log`  
**Testing**: xUnit + Moq (existing); `Microsoft.AspNetCore.Mvc.Testing` for integration tests  
**Target Platform**: ASP.NET Core 8 / Blazor Server (Linux/Windows/macOS)  
**Project Type**: Web application (Blazor Server)  
**Performance Goals**: Log writes must not block the Blazor Server render thread (Serilog's async buffering handles this)  
**Constraints**: Logging failure must be non-fatal; no stack trace in HTTP responses; no hardcoded paths  
**Scale/Scope**: Single-server deployment; daily rotation; 31-day retention default
